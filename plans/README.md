## Direction options

- Add an async job/session API for long generation runs. The frontend simulates progress while waiting for a single POST response (`frontend/script.js:131-162`), and the backend performs scrape, generation, and TTS in one request (`main.py:134-187`). This would improve reliability for slow TTS/model runs, but it is a larger product/API change than the safety fixes above.
- Add optional cleanup/retention controls for generated audio. Audio is written under `/tmp/podcast_audio` (`tts_engine.py:36`, `tts_engine.py:175-176`) and only per-line WAV files are cleaned up (`tts_engine.py:179-184`). A retention job or TTL would matter if the Space receives steady traffic.

## Findings considered and rejected

- CORS wildcard in `main.py:48-53`: not prioritized because the API does not use cookies or browser credentials, and `allow_credentials` is not enabled.
- Frontend `innerHTML` in `frontend/script.js:245-248`: not reported as a security finding because dialogue text is escaped through `escapeHtml` before insertion; future changes should preserve that escaping.
- Unused imports such as `JSONResponse`, `Request`, `Optional`, and `tempfile`: minor cleanup only, not worth a dedicated plan unless touched during nearby work.
- Synchronous work inside the FastAPI endpoint: real performance concern, but lower leverage than SSRF, file serving, and test baseline for this small app. Consider it when designing an async job/session API.
